越是關鍵選擇，大腦越容易短路，犯最低級、最不該犯的錯。

它不是學術模型，是從生存場景裡煉出來的 **「實戰保命框架」**。

後來無論是職場任務、專案立項、重要選擇，

**這不是給天才用的，是給每個不想再犯蠢、不想栽在小細節上的人。**

## 📝 防蠢 SOP 自我檢查表

### 一、復讀機模式（校準）
- [ ] 我已經用自己的話，把對方的需求重述了一遍。
- [ ] 我確認了關鍵字的定義（例如「盡快」是指今天下班前，還是明天早上？）。

### 二、三路推演（拒絕單選題）
- [ ] 方案 A（保守）：我寫出了最安全、一定做得到的方案。
- [ ] 方案 B（標準）：我寫出了資源適中、符合預期的方案。
- [ ] 方案 C（突破）：我寫出了如果沒限制、願意冒險的最佳方案。

### 三、找碴視角（紅隊測試）
- [ ] 我站在反對者角度，攻擊了自己的方案。
- [ ] 我找出了最脆弱的環節，並準備了備案。

### 四、外行驗證（價值轉譯）
- [ ] 我可以用三句話，讓完全不懂的人聽懂我的方案。
- [ ] 我講的是「結果」和「好處」，不是「過程」。

## 📌 真實填寫範例（一看就會）

### 範例1：職場需求確認

| 步驟 | 填寫內容 |
|------|----------|
| **復讀機模式** | 老闆說「盡快做一份競品分析」。我的理解：今天下班前給出核心差異點。確認「盡快」是指今天下班前，不是明天。 |
| **三路推演** | A（保守）：只做前三大競品，簡報形式。B（標準）：五家競品，附數據表格。C（突破）：十家競品，加用戶評價摘錄。 |
| **找碴視角** | 最脆弱環節：數據來源可能不準。備案：先查兩家信得過的數據源，並在報告中註明數據日期。 |
| **外行驗證** | 「給老闆三個競品的主要差異，讓他五分鐘內看懂我們該往哪走。」 |

### 範例2：專案立項決策

| 步驟 | 填寫內容 |
|------|----------|
| **復讀機模式** | 客戶說「希望產品有質感」。我的理解：簡約風格，高級感。確認「質感」是指極簡風，不要花俏。 |
| **三路推演** | A（保守）：現有模板小改。B（標準）：重新設計視覺，保留原框架。C（突破）：從頭打造品牌視覺系統。 |
| **找碴視角** | 最脆弱環節：突破方案可能超時。備案：預留兩週緩衝，先出核心頁面，其他疊代。 |
| **外行驗證** | 「讓客戶打開網頁第一眼就覺得『這東西很貴、很厲害』。」 |

## 🤝 為什麼有效？

這套SOP不是理論，是在極限環境下被逼出來的：
- **強迫校準**：先確認資訊，避免瞎衝
- **強迫多選**：至少三個方案，不是賭博
- **強迫自我攻擊**：先補洞，別等人來罵
- **強迫講人話**：確認你真的想清楚了

理論精度可能不如1000層推演，但**實戰存活率極高**。
